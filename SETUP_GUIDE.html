☕ Carroll Street Café POS

Complete setup guide · Supabase + Hostinger + Telegram bot with voice

1Supabase — Create your database

Go to supabase.com → New project → Choose a name and password.

Once created, go to SQL Editor and run the contents of supabase_setup.sql. This creates the sales table and enables realtime.

Then go to Settings → API and copy:
• Project URL (looks like https://xxxxx.supabase.co)
• anon public key (long string starting with eyJ…)

2Hostinger — Deploy the web POS

Log in to Hostinger → File Manager → go to your subdomain's public_html folder.

Upload index.html. That's it — your POS is live at your subdomain.

💡 Share the subdomain URL with all 4 cashiers. They open it in their phone browser — no app install needed. Bookmark it on the home screen for fastest access.

3First launch — connect Supabase

Open your subdomain URL. You'll see a one-time setup screen. Enter your Supabase URL and anon key. These are saved locally on that device — each cashier only needs to do this once.

After connecting, the POS loads with the full Carroll Street Café menu ready to go.

4Telegram Bot — Create it

On Telegram, message @BotFather:

/newbot
→ Name: Carroll Street Café
→ Username: carrollstreetcafe_bot (or any available name)
→ Copy the BOT TOKEN it gives you

Then message @userinfobot to get your personal or group Chat ID.

💡 Create a Telegram group with all 4 cashiers + yourself, add the bot to the group, then get the group's Chat ID to allow all of them at once.

5OpenAI — Voice transcription (free)

Go to platform.openai.com → API keys → Create key.

The Whisper API costs about $0.006/minute of audio. For short voice orders (~10 seconds each), 100 voice orders = ~$0.01. Essentially free.

6Railway — Deploy the bot server (free)

Go to railway.app → New Project → Deploy from GitHub repo.

Push your bot.js and package.json to a GitHub repo, then connect it to Railway.

In Railway → your service → Variables, add these env vars:

BOT_TOKEN = (from BotFather)
SUPABASE_URL = https://xxxxx.supabase.co
SUPABASE_KEY = eyJ…(anon key)
OPENAI_API_KEY = sk-…
ALLOWED_CHAT_IDS = 123456789,987654321 (comma-separated)
PORT = 3000

Railway will give you a public URL like https://your-app.up.railway.app

7Connect bot to Telegram webhook

In your browser, visit this URL (replace values):

https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook?url=https://<YOUR_RAILWAY_URL>/webhook

You should see: {"ok":true,"result":true}

8Using the Telegram bot

Any cashier in the allowed chat can:

sell 2 lattes, 1 avocado toast, 3 sodas   ← type an order
report                                      ← today's totals
report week                                 ← this week
staff Amara                                 ← set your name
menu                                        ← see all items

Or send a voice note saying the order — the bot transcribes it and asks for confirmation before saving.

💡 Voice note flow: speak → bot shows what it heard → reply "yes" to save. All sales from Telegram appear instantly in the web POS reports marked with 📱 Telegram.

9You're live!

All 4 cashiers can now:

✅ Use the web POS on their phones at your-subdomain.com
✅ Log sales via Telegram by typing or voice
✅ See live reports updating in real time
✅ Export sales to CSV anytime
✅ Add/edit menu items from the Manage tab

💡 Total ongoing cost: $0/month. Supabase free tier: 500MB storage, 2GB bandwidth. Railway free tier: 500 hours/month. Hostinger: your existing plan.
